API for Processing Connected Car Data (Client: Top 3 global automaker)

The project involved creating a GraphQL API to retrieve automobile sensor data from the past 90 days, facilitating advanced analyses such as pothole detection. Substantial cost savings of 35% were achieved by optimizing storage by configuring AWS S3 policies based on usage, utilizing Terraform and CI/CD pipelines. Additionally, Spark jobs were scripted to efficiently process generated parquet files, leading to the construction of API payloads for 13 distinct data requests, enhancing overall data processing capabilities.

The project involved constructing predictive models in R to forecast the influence of trade promotions on sales and optimize related strategies. Additionally, the migration of intermediate storage from SAS Grid to AWS S3 not only reduced license costs but also achieved 99% data durability. Analytics for Efficient Use of Crash Prototypes (Client: Top 5 global automaker)

The project involved the creation of a threshold-based classifier to identify crucial crash tests, leading to 73% optimization in the total number of tests. Data mining tools were implemented to process automobile sensor data, extracting 5+ qualitative features for synthesizing 108 future program test scenarios for non-critical tests with an impressive 86% accuracy, showcasing the effectiveness of the developed tools in predictive analysis within the context of global automaker crash testing.
