API for Processing Connected Car Data

Project Overview

In the era of modern vehicles equipped with an intricate web of sensors, extracting and decoding the raw data emitted by these sensors poses a significant challenge. This challenge is precisely what the API project addressesâ€”a cutting-edge API-based service designed to seamlessly extract Vehicle CAN data from end to end.

The primary purpose is to provide a streamlined solution for decoding and extracting high-volume data. As vehicles generate vast amounts of data through Electronic Control Units (ECUs) and Controller Area Network (CAN) networks, decoding this information becomes essential for various applications. The API decodes raw data and extracts decoded data from both hot and cold S3 storage, making it accessible to end-users.

The product encompasses a suite of GraphQL-based APIs designed to extract vehicle data seamlessly. The APIs serve diverse purposes, from token generation for authorization using Cognito credentials to extracting data by Vehicle Identification Number (VIN) with varying time frames. The APIs cater to various use cases, including extracting data hourly and supporting real-time data profiling. Payload parameters are meticulously defined, covering VINs, dispatch model types, start and end dates, and options for including GPS data. The output data schema adheres to the Apache Parquet format, optimizing data storage and retrieval with efficient compression and encoding schemes.
